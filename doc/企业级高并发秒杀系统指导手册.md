# 高并发秒杀系统项目指南（Spring Boot + React）

## 1. 项目目标
本项目旨在构建一个最小可用的高并发秒杀系统，用于求职简历展示，具备以下核心功能：

- 前端商品展示和秒杀按钮（React）
- 后端秒杀接口（Spring Boot）
- Redis 原子库存扣减（Lua 脚本）
- MQ 异步下单（RabbitMQ）
- MySQL 持久化订单
- 支持高并发（万级 QPS）
- 企业级开发与交付最佳实践

---

## 2. 本地开发环境（MacBook Air M4）
### 2.1 必装软件
- **JDK 17+**（用于 Spring Boot）
- **Node.js 18+**（用于 React）
- **npm / yarn**（Node 包管理器）
- **Git**（版本管理）
- **Docker Desktop**（运行中间件容器）
- **IDE**：IntelliJ IDEA（Java）、VS Code（React 前端）

### 2.2 可选工具
- Postman 或 Insomnia（接口调试）
- DBeaver / TablePlus（数据库管理）

### 2.3 Docker 容器
本地开发只用 Docker 运行中间件，不需要把应用打包到 Docker：
- MySQL 8
- Redis 6/7
- RabbitMQ

```yaml
# docker-compose.dev.yml
version: '3'
services:
  mysql:
    image: mysql:8
    environment:
      MYSQL_ROOT_PASSWORD: 123456
      MYSQL_DATABASE: seckill
    ports:
      - "3306:3306"
    volumes:
      - ./mysql_data:/var/lib/mysql
  redis:
    image: redis:7
    ports:
      - "6379:6379"
  rabbitmq:
    image: rabbitmq:3-management
    ports:
      - "5672:5672"
      - "15672:15672"
```

### 2.4 本地调试方式
- 启动 Docker 容器：`docker-compose -f docker-compose.dev.yml up -d`
- 修改 Spring Boot `application-dev.yml` 指向 `localhost` 的 MySQL/Redis/RabbitMQ
- IDE 启动 Spring Boot 应用
- React 本地运行 `npm run dev`，访问前端页面

## 4. Dockerfile 构建
### 4.1 后端 Dockerfile
```dockerfile
# Dockerfile.backend
FROM eclipse-temurin:17-jdk AS build
WORKDIR /app
COPY backend/pom.xml ./
COPY backend/src ./src
RUN ./mvnw clean package -DskipTests

FROM eclipse-temurin:17-jre
WORKDIR /app
COPY --from=build /app/target/backend.jar ./backend.jar
EXPOSE 8080
ENTRYPOINT ["java","-jar","backend.jar"]
```

### 4.2 前端 Dockerfile
```dockerfile
# Dockerfile.frontend
FROM node:18 AS build
WORKDIR /app
COPY frontend/package*.json ./
RUN npm install
COPY frontend/src ./src
RUN npm run build

FROM nginx:alpine
COPY --from=build /app/dist /usr/share/nginx/html
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

### 4.3 构建与推送
```bash
# 构建后端镜像
docker build -t my-registry/shopping-backend:v1 ./docker/Dockerfile.backend
# 构建前端镜像
docker build -t my-registry/shopping-frontend:v1 ./docker/Dockerfile.frontend
# 推送镜像
docker push my-registry/shopping-backend:v1
docker push my-registry/shopping-frontend:v1
```

## 5. 企业级部署
### 5.1 配置分离
- 本地：`application-dev.yml`
- 生产：使用环境变量或 K8s ConfigMap/Secret
```yaml
# docker-compose.prod.yml
services:
  backend:
    image: my-registry/shopping-backend:v1
    environment:
      - SPRING_DATASOURCE_URL=jdbc:mysql://prod-db.rds.aliyuncs.com:3306/seckill
      - SPRING_REDIS_HOST=prod-redis.redis.aliyuncs.com
      - SPRING_RABBITMQ_HOST=prod-rabbitmq.cloud
    ports:
      - "8080:8080"
  frontend:
    image: my-registry/shopping-frontend:v1
    ports:
      - "80:80"
```

### 5.2 生产架构

## 6. 额外注意事项
- **缓存问题**：缓存穿透（BloomFilter）、缓存击穿（互斥锁）、缓存雪崩（随机过期）
- **限流**：用户/IP/接口限流 + 验证码 + 秒杀开关
- **库存扣减**：Redis Lua 脚本原子操作 + MQ 异步下单
- **分布式锁**：Redisson 或 Redis setNX 解决库存冲突
- **日志监控**：Spring Boot Actuator + ELK 或 Prometheus/Grafana
- **前端优化**：静态页面 + CDN，秒杀按钮 JS 控制亮灭

---

## 7. 总结
1. **本地开发**：IDE + Docker 中间件
2. **开发调试**：Lua 脚本、高并发测试、Git 管理
3. **构建交付**：多阶段 Dockerfile 生成镜像，推送私服
4. **生产部署**：K8s 或 Docker Compose，连接生产级中间件

> 遵循关注点分离与环境一致性原则，使本项目既可用于求职展示，又可模拟企业级高并发秒杀系统落地实践。
